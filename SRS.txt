1. Introduction 
1.1 Purpose 
This document defines the functional and non-functional requirements for a web-based Learning 
Management Platform that integrates with Moodle LMS using REST APIs while providing a custom 
frontend and backend system. The platform will allow students, parents, schools, and 
administrators to manage courses, enrollments, payments, progress tracking, and reporting. 
1.2 Scope 
The system will provide: 
ï‚· Course marketplace 
ï‚· Student learning dashboard 
ï‚· Parent monitoring dashboard 
ï‚· School bulk purchase and licensing 
ï‚· Payment processing via Pakistani gateways 
ï‚· Single Sign-On with Moodle 
ï‚· Reporting and analytics 
ï‚· Forms management 
ï‚· Notifications 
ï‚· User management 
ï‚· Secure authentication 
Moodle will act as the learning engine while the custom system manages business logic.
